# Validation Contract

Nenhuma mudança grande entra sem prova.

Obrigatório:
- testes relevantes rodando
- diff explicado em linguagem humana
- impacto por serviço
- plano de rollback
- observabilidade mínima
- evidência de que o comportamento anterior foi preservado onde necessário

Defina claramente:
- o que deve falhar
- o que deve passar
- qual é a fonte da verdade
