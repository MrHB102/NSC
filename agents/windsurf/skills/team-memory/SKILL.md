# Team Memory

        ## Quando usar
        Quando a história da equipe muda a interpretação do código.

        ## Registrar
        - decisões antigas
        - tentativas falhas
        - motivos de arquitetura
        - áreas de responsabilidade
        - preferências de revisão

        ## Regra
        Use esse skill como opcional; não invente memória.
