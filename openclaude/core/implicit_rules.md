# Implicit Rules

Regras que o repositório segue sem dizer:

- convenções de nome
- ordem de chamadas
- padrões de commit
- formatos esperados de payload
- permissões informais
- invariantes de negócio
- shortcuts da equipe
- casos onde "funciona" mas não está documentado

O trabalho do agente é transformar regras implícitas em regras explícitas.
