# NSC — NonSlopContext

Persistent context and governance for autonomous coding agents.

## What is NSC?

NonSlopContext (NSC) is a meta-context and governance framework designed to reduce architectural entropy, unsafe autonomous behavior and AI-generated software degradation ("AI slop") in coding agents.

NSC provides:

- persistent contextual layers
- architectural memory
- governance rules
- risk-aware execution
- incremental planning
- anti-slop guardrails
- enterprise-oriented workflows

Supported agents:

- Claude Code
- Codex
- OpenClaude / OpenCode
- Cursor
- Windsurf
- Continue
- Aider
- OpenHands

---

# Important Disclaimer

NSC reduces risk.

NSC does NOT completely eliminate:

- hallucinations
- architectural mistakes
- unsafe refactors
- implicit business logic breakage
- hidden dependency issues
- context degradation

Human supervision is still required.

NSC is a governance layer, not a guarantee of correctness.

---

# Human + LLM Assisted Creation

NSC was created with assistance from Large Language Models (LLMs).

However:

- the architectural direction
- framework concepts
- governance model
- contextual engineering approach
- system design
- operational philosophy

were primarily designed and curated by humans.

LLMs were used mainly as acceleration and implementation assistance tools.

# How NSC Works

NSC introduces multiple contextual layers:

```txt
Business Context
↓
Architectural Context
↓
Risk Analysis
↓
Historical Context
↓
Implicit Rules
↓
Incremental Planning
↓
Safe Execution
↓
Validation
```

Instead of:

```txt
Prompt → Massive Autonomous Refactor → Chaos
```

---

# Main Concepts

## Context Engineering

NSC focuses on preserving:

- architectural consistency
- business constraints
- implicit organizational knowledge
- system boundaries
- operational safety

---

## Meta-Context

The framework injects:

- business goals
- architectural memory
- risk maps
- blast radius analysis
- dependency awareness
- incremental execution rules

into coding agents.

---

## Anti-Slop Execution

NSC attempts to prevent:

- excessive refactors
- unnecessary abstractions
- agentic overreach
- unsafe multi-file modifications
- hidden dependency corruption

---

# Enterprise Focus

NSC was designed for:

- monorepos
- legacy systems
- hybrid infrastructure
- large codebases
- multiple services
- implicit organizational rules
- high-autonomy agents

---

# Philosophy

NSC does not try to make agents "more intelligent".

NSC attempts to make autonomous systems:

- safer
- more contextual
- more constrained
- more auditable
- more operationally predictable
